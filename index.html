Rekursion / Dynamic Programming

  • Bestimmung eines Rekursionsankers - Welche Eingabe führt zu einem bestimmbaren Ergebnis
  • Aufbrechen in Unterprobleme → Welches Problem lässt sich am einfachsten lösen? (meistens der Rekursionsanker)

DepthFirstSearch (DFS)

  • Deutsch: Tiefensuche
  • Nutzt einen Stack → Untersucht einen Pfad “blind” bis ein Ende erreicht wird

BreathFirstSearch (BFS)

  • Deutsch: Breitensuche
  • Nutzt eine Queue → Untersucht jeden Pfad bis ein Ende erreicht wird

Two-Pointer-Approach

  • Ein "schneller" und ein "langsamer" Pointer
  • Je nach Fall werden die Werte der Pointer verschoben
  • While-Schleife (meistens solange links ≤ rechts)

Brute-Force-Approach

  • Geht jede mögliche Kombination der Daten durch
  • Meistens die einfachste Strategie, aber auch am ressourcenaufwendigsten
